# ⚡ Proteus 8.0 LED电路 - 快速开始指南

## 🎯 30秒快速开始

### 步骤1：打开项目
```
1. 启动 Proteus 8.0
2. File → Open Project
3. 选择 LED_Circuit.pdsprj
4. 点击"打开"
```

### 步骤2：运行仿真
```
1. 点击工具栏的绿色 ▶ 按钮（或按 Ctrl+R）
2. 观察LED发光效果
3. 仿真运行1秒后自动停止
```

### 步骤3：查看结果
```
1. 使用 Probe 工具（鼠标悬停在连线上）
2. 查看节点电压和电流值
3. 预期值见下表
```

## 🔧 常用操作

### 查看电压
1. 在菜单中选择 `Simulation` → `Voltage Probe`
2. 点击要测量的电路节点
3. 窗口底部显示实时电压值

### 查看电流
1. 在菜单中选择 `Simulation` → `Current Probe`
2. 点击要测量的导线
3. 显示通过该处的电流值

### 修改参数
1. **修改电源电压**：双击VCC → 输入新的电压值
2. **修改电阻值**：双击R1 → 输入新的电阻值（如180、220、270等）
3. **修改LED类型**：双击D1 → 从库中选择其他LED

### 暂停/继续仿真
1. 点击工具栏的 ⏸ 按钮暂停
2. 点击 ▶ 按钮继续
3. 按 Ctrl+R 快速切换

### 重置仿真
1. 点击工具栏的 ⏹ 按钮停止
2. 仿真自动复位到初始状态

## 🎓 学习任务

### 任务1：验证欧姆定律
```
目标：测量电路中的电压和电流，验证 V = I × R

步骤：
1. 运行仿真
2. 用 Voltage Probe 测量 R1 两端电压
3. 用 Current Probe 测量电路电流
4. 计算：电压 / 电流 应该约等于 220Ω

预期结果：
3.64V / 13.6mA ≈ 268Ω (接近220Ω，偏差来自LED压降)
```

### 任务2：调整LED亮度
```
目标：通过改变电阻值改变LED亮度

实验：
1. 将 R1 改为 150Ω → LED变亮（电流增加）
2. 将 R1 改为 330Ω → LED变暗（电流减少）
3. 记录不同电阻对应的电流和LED亮度

注意：
- LED最大安全电流为 30mA
- 不要设置电阻 < 100Ω（会烧毁LED）
```

### 任务3：功率消耗计算
```
目标：计算各元件的功率消耗

方法：
1. 测量电流 I = 13.6mA
2. 测量 R1 上的压降 V_R = 3.64V
3. 测量 LED 上的压降 V_LED = 2.0V

计算：
- R1 功率 = V_R × I = 3.64V × 13.6mA = 49.5mW ✓
- LED 功率 = V_LED × I = 2.0V × 13.6mA = 27.2mW ✓
- 总功率 = 5V × 13.6mA = 68mW ✓
```

### 任务4：极性实验
```
目标：理解 LED 的极性要求

实验：
1. 运行原始仿真（LED应该亮）
2. 在电路编辑中反接 LED（交换两条引脚）
3. 运行仿真 → LED 不亮
4. 观察：LED 反接时电流为 0

结论：
LED 只允许单向导通，反接时完全断路。
```

## ⚠️ 常见问题排查

### 问题1：LED不亮
**原因检查清单：**
- [ ] LED 极性是否正确？（长腿为正）
- [ ] 电源连接是否正确？
- [ ] 电阻是否过大（>1kΩ）？
- [ ] 仿真是否真的在运行？（查看状态栏）

**解决方案：**
1. 重新检查 LED 极性
2. 将 R1 改为 220Ω（标���值）
3. 停止仿真后点 ▶ 重新开始

### 问题2：显示"Simulation Failed"
**可能原因：**
- 元件参数设置错误
- 电路有短路
- 文件损坏

**解决方案：**
1. 点击 `Tools` → `Simulation` → `Reset All`
2. 关闭并重新打开文件
3. 如果仍有问题，参考 README.md 手动创建电路

### 问题3：仿真速度太慢
**优化方法：**
1. 增加时间步长（在设置中改为 1ms）
2. 关闭不必要的显示功能（LED 动画、网格等）
3. 关闭其他应用程序释放内存
