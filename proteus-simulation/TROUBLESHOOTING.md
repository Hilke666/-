# 🔧 Proteus 8.0 LED 电路 - 故障排查指南

## 问题1：LED不亮

### 症状
- 仿真运行，但 LED 没有显示发光（应该是红色）
- 或 LED 显示为暗色

### 检查清单

#### ✓ 检查1：LED 极性
```
正确：LED 长腿(+) → R1 → VCC
      LED 短腿(-) → GND

错误：LED 长腿(+) → GND
      LED 短腿(-) → VCC
```

**修复方法：**
1. 双击 D1 (LED)
2. 检查引脚标记：A=阳极(+), K=阴极(-)
3. 如果接反，右键选择 "Rotate 180°" 或重新连接

#### ✓ 检查2：电源连接
```
正确电路：
  5V ──┬──[220Ω]──┬──[LED]──┬── GND
       └─ VCC     └─ D1   └─ GND

检查要点：
- VCC 是否连接到 R1 的一端？
- R1 的另一端是否连接到 LED 正极(A)？
- LED 负极(K) 是否连接到 GND？
```

**修复方法：**
1. 在原理图中点击每条连线
2. 确认连接点显示蓝色（连接成功）
3. 红色虚线表示连接失败

#### ✓ 检查3：电阻值过大
```
如果 R1 > 500Ω，LED 电流会太小而看不亮。

标准选择：
- 150Ω → 很亮（20mA）
- 220Ω → 中等亮度（13.6mA）← 推荐
- 330Ω → 较暗（10.2mA）
```

**修复方法：**
1. 双击 R1
2. 将值改为 "220" (Ω)
3. 确认 Tolerance 为 "5%"

#### ✓ 检查4：仿真是否真的在运行
```
检查指标：
- 窗口标题栏显示 "[RUNNING]"
- 工具栏中 ▶ 按钮是灰色（表示正在运行）
- ⏸ 按钮是彩色（可以点击暂停）
- 仿真时间计时器在增加
```

**修复方法：**
1. 如果没有运行，点击 ▶ 开始仿真
2. 等待至少 0.1 秒让仿真稳定
3. 如果 1 秒后自动停止，再点一次 ▶

#### ✓ 检查5：LED 模型是否正确
```
验证方法：
1. 用 Voltage Probe 测量 D1 两端电压
2. 应该显示约 2.0V
3. 如果显示 0V 或 5V，说明 LED 模型可能有问题
```

**修复方法：**
1. 删除现有 LED
2. 从库重新添加：`Pick Device` → 搜索 "LED" → 选择 "LED-RED"
3. 重新连接导线

## 问题2："Simulation Error" 或 "Failed to Converge"

### 症状
- 仿真无法启动
- 显示错误信息
- 仿真崩溃或卡住

### 解决方案

#### 方案1：重置仿真引擎
```
1. 点击 Simulation → Simulation Settings
2. 点击 "Reset" 按钮
3. 确认所有参数（见下表）
4. 点击 "OK" 重新开始
```

| 参数 | 值 | 说明 |
|------|-----|------|
| Analysis Type | Transient | 瞬态分析 |
| Start Time | 0s | 开始时间 |
| End Time | 1s | 结束时间 |
| Time Step | 0.1ms | 时间步长 |
| Tolerance | 1e-6 | 收敛容限 |

#### 方案2：检查元件参数
```
确保每个元件都有正确的参数：

□ VCC:
  - 电压值：5
  - 单位：V
  - 无源特性：正常

□ R1:
  - 阻值：220
  - 单位：Ω
  - 功率：1/4W

□ D1:
  - 类型：LED
  - 颜色：Red
  - 型号：通用红色 LED

□ GND:
  - 引用：GND
  - 电位：0V
```

#### 方案3：检查导线连接
```
症状：虽然看起来已连接，但仿真报告"浮节点"（断开）

修复步骤：
1. 删除所有导线（Edit → Select All → Delete）
2. 使用 Wire 工具重新连接
3. 确保每个连接点都显示小圆点（表示连接）
4. 重新开始仿真
```

#### 方案4：重新加载文件
```
如果上述方法都不行：

1. File → Close 关闭项目（不保存）
2. File → Open Project 重新打开
3. 仿真应该恢复正常

如果仍然失败：
1. 参考 README.md 手动重新创建电路
2. 从头添加元件和连线
```

## 问题3：测量结果与预期不符

### 症状
- 仿真运行，但测量到的电压/电流与计算不符
- 例如：测得电流 20mA 而不是 13.6mA

### 原因分析

#### ���能原因1：LED 参数设置错误
```
症状：电流异常高（> 20mA）
原因：LED 的正向压降(Vf) 设置过小

修复：
1. 双击 D1
2. 查看 Vf 参数（应该是 2.0V ± 0.2V）
3. 如果不对，重新选择 LED 型号
```

#### 可能原因2：电源电压不是 5V
```
症状：电流和功率都异常
原因：VCC 设置错误

修复：
1. 双击 VCC
2. 确认值为 5（单位 V）
3. 如果是其他值，改为 5
```

#### 可能原因3：存在额外的电路路径
```
症状：电流低于预期
原因：可能有断路或额外的高阻抗路径

修复：
1. 仔细检查原理图
2. 使用 Voltage Probe 逐点测量
3. 确认没有多余的元件或错误的连接
```

#### 可能原因4：仿真还没有稳定
```
症状：测量值在变化（特别是早期）
原因：瞬态还未结束

修复：
1. 等待至少 100ms 让电路稳定
2. 在 t > 0.5s 的时间点进行测量
3. 使用 "Pause" 功能在特定时间停止
```

## 问题4：LED 太亮或太暗

### LED 太亮（接近最大亮度）

**原因**：电流过大（> 20mA）

**解决方案**：增加电阻值
```
当前：R1 = 150Ω → I ≈ 20mA
改为：R1 = 220Ω → I ≈ 13.6mA
或者：R1 = 330Ω → I ≈ 10.2mA
```

**操作步骤**：
1. 双击 R1
2. 将 "Value" 改为 220（或更大）
3. 重新运行仿真

### LED 太暗

**原因**：电流过小（< 10mA）

**解决方案**：减小电阻值
```
当前：R1 = 470Ω → I ≈ 6.4mA （太暗）
改为：R1 = 220Ω → I ≈ 13.6mA ✓ 中等
或者：R1 = 150Ω → I ≈ 20mA （很亮）
```

**注意**：
- 不要设置 R1 < 100Ω（可能烧毁 LED）
- LED 最大安全电流为 30mA
- 220Ω 是最平衡的选择

## 问题5：无法保存或打开文件

### 症状
- 保存时提示"Permission Denied"
- 打开文件时提示"File Not Found"
- 文件损坏无法打开

### 解决方案

#### 权限问题
```
1. 关闭 Proteus
2. 右键点击文件夹 → 属性 → 安全
3. 确保当前用户有读写权限
4. 重新打开 Proteus
```

#### 文件不存在
```
1. 检查文件路径是否正确
2. 确认文件名拼写无误
3. 使用绝对路径而不是相对路径
4. 确保文件未被删除或移动
```

#### 文件损坏
```
1. 检查 .bak 备份文件
2. 如果有备份，尝试打开它
3. 如果无备份，需要重新创建电路
4. 建议手动重新添加元件
```

## 问题6："Warning: LED reverse biased"

### 症状
- 仿真运行但显示警告
- LED 完全不亮
- 消息："反向偏置"

### 原因
LED 的极性接反了。

### 立即��复
```
1. 右键点击 D1 (LED)
2. 选择 "Rotate 180°"
3. 或者点击 D1 然后按 "R" 旋转
4. 重新运行仿真
```

### 验证
```
正确的连接应该是：
- LED 的长腿 (阳极 A) → 连接到 R1 的输出
- LED 的短腿 (阴极 K) → 连接到 GND
```

## 快速诊断树

```
LED 不亮？
 ├─ 仿真在运行吗？
 │  ├─ 否 → 点击 ▶ 启动
 │  └─ 是 ↓
 ├─ LED 极性正确吗？
 │  ├─ 否 → 旋转 LED 180°
 │  └─ 是 ↓
 ├─ 用 Voltage Probe 测量
 │  ├─ LED 两端 ≈ 2V → LED 正常，可能显示问题
 │  ├─ LED 两端 = 0V → 可能断路
 │  ├─ LED 两端 = 5V → LED 反向接
 │  └─ LED 两端 > 5V → 参数错误
 └─ 检查电阻值
    ├─ > 500Ω → 改为 220Ω
    └─ 电路重新连接
```
